# LoopNet v0.2 — Data-first plan

**Theme:** Real trajectories from LoopGym runs, merged with v0.1 seed, re-published on Hugging Face.

## Goal

Move from 100% synthetic seed to a corpus that includes **captured loop runs** — still schema-valid, still ≥40% failures at corpus level, ready for ML and replay.

## Capture workflow

```bash
pip install loopgym

# One environment
loopgym capture loopbench/code-repair-v1 \
  --task-ids cr-001,cr-002,cr-003 \
  --seeds 0,1,2,3,4 \
  --failure-seeds 2,3,4 \
  -o loopnet/data/captured/code-repair-v1.jsonl

loopgym capture loopbench/research-synthesis-v1 \
  --task-ids rs-001,rs-002 \
  --seeds 0,1,2,3,4 \
  -o loopnet/data/captured/research-synthesis-v1.jsonl

loopgym capture loopbench/multi-agent-debate-v1 \
  --task-ids ma-001,ma-002 \
  --seeds 0,1,2,3,4 \
  -o loopnet/data/captured/multi-agent-debate-v1.jsonl
```

Validate captures (per file):

```bash
cd loopnet
python scripts/validate_record.py data/captured/*.jsonl --skip-corpus-policy
```

Build v0.2 corpus:

```bash
python scripts/merge_corpus.py --validate
```

Upload (after HF secret configured):

```bash
# Update upload-hf.yml default repo_id to KanakMalpani/loopnet-v0.2
python scripts/upload_hf.py --export-parquet --upload \
  --repo-id KanakMalpani/loopnet-v0.2 \
  --jsonl data/v0.2/records.jsonl
```
